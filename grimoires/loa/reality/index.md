## What TurnTrace actually is

A **data-loop and evaluation harness** for a simulator-based trading-card-game agent
(Kaggle Pokémon TCG AI Battle / `cabt` simulator). It runs real matches, seals
immutable run artifacts with provenance, aggregates results into an append-only
ledger, and enforces a **claim ceiling** so no unearned strength claim escapes.

Maturity: through **Cycle-007**; standing claim ceiling **Rung 2 — "beats random-legal"**
(`docs/claim-ceiling.md`). Not bootstrap-only (the README is stale; see `../drift-report.md`).

## Quick stats

- 21 Python files, ~5,689 LOC, **stdlib-only** app code (NFR-7).
- Pipelines: `probe → run_eval → aggregate → {delta,dispersion,evidence,failure,e2e,replay}`.
- Two agents: `random_legal` (RNG floor), `scripted_baseline` (deterministic).
- Frozen regimes: v001, v002, v003. Ledger: 3 data rows.
- Sim seed is **uncontrolled** (`seed_controlled=false`) → audit-trail reproducibility, not byte-identical.
