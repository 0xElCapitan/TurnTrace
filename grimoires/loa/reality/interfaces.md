# Interfaces — Extracted Reality

> External integration boundaries. Generated by `/ride`.

## The cabt simulator boundary (the single blast radius)

TurnTrace integrates the local **cabt** simulator (the `cg` package;
kaggle-environments 1.14.10). `sim/adapter.py` and `sim/probe.py` are the **only**
two modules that import `cabt`/`cg` directly (`sim/adapter.py:73`, `sim/probe.py:76`).
This is enforced by `tests/test_import_direction.py` (AST lint).

`cg.game` surface used: `battle_start(deck0, deck1) -> (obs_dict, StartData)`,
`battle_select(selection) -> obs_dict`, `battle_finish()`. **No seed parameter is
accepted** → `seed_controlled=false` (`sim/probe.py:393`, `sim/adapter.py:243`).

### Harness-internal port (what the rest of the harness sees)

`SimAdapter` exposes: `start_match` / `step` / `finish` / `legal_options` /
`capabilities` + Competition-Data-free observation views. Callers outside `sim/`
depend on THIS contract, never on cabt internals. If the live API diverges, only
files inside `sim/` change (SDD §5.1).

### Capability discovery

`sim/probe.py` runs real match(es) and writes `sim/capabilities.json` +
`sim/README.md`. `SimAdapter.capabilities()` reads that file, or falls back to
conservative defaults (`local_match_runnable="unconfirmed"`, all flags off) if
absent (`sim/adapter.py:109-133`). Last probe (2026-06-18): match runnable,
~92 matches/sec, ~56 decisions/match (`sim/capabilities.json`).

## Agent submission contract (Kaggle/cabt)

`agent(obs_dict) -> list[int]` — both runtime agents implement it
(`random_legal.agent`, `scripted_baseline.agent`). The initial deck-selection
step (`select is None`) is the runner's responsibility, not the agent's. A second
**in-process** contract `select(n_options, min_count, max_count, rng)` carries
only the legal-offer shape (counts, never option types) so no scoring can leak
onto the runtime path (SDD §1.6).

## Git boundary

`eval/run_eval.py` shells out to `git rev-parse HEAD` / `git status --porcelain`
for provenance (`:71-77`); failures degrade to `"unknown"`/`"false"`, never
crash. `eval/hygiene_check.py` reads `git diff --cached --name-only` as a
pre-commit guard.

## Competition-Data containment (CC-1/CC-2, ESP-1)

Raw card IDs/names/deck lists are **Competition Data**: local-only, git-ignored,
never written to a tracked artifact. The boundary is enforced three ways:
1. Adapter views emit only counts, OptionType enums, and SHA-256 digests of
   card-id lists — never raw IDs (`sim/adapter.py:15-19`, `_digest` `:51`).
2. `eval/hygiene_check.py` blocks staging of `cg/`, `deck.csv`,
   `grimoires/loa/context/`, raw run trees, PDFs, card-data CSVs.
3. `analysis/evidence_summary.py` validator hard-fails (exit 3) on forbidden
   fields/values/words leaking into a summary.

## No network / no external services

App code is stdlib-only with **no network calls** (CC-3 / NFR-8). No DB, no HTTP
client, no cloud SDK. The only "external" surfaces are the local cabt library,
the local filesystem (run dirs), and `git`.
