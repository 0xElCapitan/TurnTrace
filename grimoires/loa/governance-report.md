## Gaps flagged for human decision

- **G-1 [LOW]** No `LICENSE`. Decide whether this repo should carry one (competition
  rules may require or forbid redistribution; Competition Data is already non-redistributable).
- **G-2 [LOW]** No `SECURITY.md`. External attack surface is minimal (stdlib-only, no
  network), so this is low priority; add a one-liner if the repo is ever published.
- **G-3 [INFO]** No `CHANGELOG.md`. The cycle-closeout + ledger pattern already provides
  an auditable history; a CHANGELOG would be additive, not corrective.
- **G-4 [INFO]** No project release tags. Run/ledger provenance (git_rev in `hashes.txt`)
  already pins each evaluation to a commit, so release tags are optional.

## Assessment

The *substance* of governance (reproducibility, provenance, claim discipline) is
present and well-engineered. The *ceremony* of OSS governance is absent and mostly
appropriate to absent for a private competition harness. None of the gaps block work;
all are human-decision items.
